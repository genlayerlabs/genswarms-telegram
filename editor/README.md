# genswarms_telegram_editor

Faithful preview renderer + generic editor shell for the Telegram rich
messages generated by `genswarms_telegram`'s `Card.to_rich_message/2`.
Dev tooling: depend on it `only: [:dev, :test]`; it has zero runtime deps
and ships no production code paths.

- `priv/preview/telegram_preview.mjs` — dependency-free browser ESM.
  `renderPreviewHtml(html, {profile: "classic"|"rich", loadMedia})`,
  `attachPreviewBehaviors(root)`.
- `priv/preview/telegram_preview.css` — Telegram-faithful styles (light/dark).
- `priv/preview/editor.html` — two-pane editor shell; hosts serve it and
  pass `?endpoint=<url>` pointing at their own validate+render endpoint.
- `priv/tags.json` — the generator↔preview contract (generated from
  `priv/preview/tags.mjs`; regenerate with `node scripts/gen-tags-json.mjs`).
- `priv/calibration/<bot-api>/` — real-client screenshots the CSS is
  reviewed against.

Host integration: serve `GenswarmsTelegramEditor.asset_dir/0` read-only and
fail boot when `supported_schema/0` differs from
`Genswarms.Telegram.Card.schema_info().version`.

Tests: `mix test` (contract vs main package) and `node --test test/*.test.mjs`
(renderer, behaviors, shell — the bare `test/` directory form fails on some
Node versions; the glob is the reliable invocation). Fixtures come from
`mix editor.fixtures` run at the repo root.
